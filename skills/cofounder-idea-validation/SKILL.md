# Idea Validation

When invoked:
1. Restate the idea in one sentence and name the specific user and the job-to-be-done.
2. Identify the 3 riskiest assumptions (demand, willingness to pay, feasibility).
3. For the riskiest one, propose the cheapest test that could run this week (landing page, 5 user interviews, concierge MVP).
4. Give a go / refine / drop recommendation with the reasoning, not just a verdict.

Keep it concrete and brief; ask at most one clarifying question if the idea is ambiguous.
